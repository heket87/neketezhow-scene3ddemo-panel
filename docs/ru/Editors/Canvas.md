# Canvas

Раздел **Canvas** отвечает за внешний вид холста, фон, тени, производительность и служебные инструменты сцены. Здесь настраивается не содержимое сцены, а то, как эта сцена отображается пользователю.

## Оформление списка быстрых ссылок

Эти параметры работают только если в разделе **Scene3D Panel Plugin** включён **Add fast links list**.

### Font color of links list

Цвет текста в списке быстрых ссылок.

### Background color of links list

Цвет фона блока со ссылками.

### Borders color of links list

Цвет рамки списка ссылок.

### Borders width of links list

Толщина рамки списка ссылок.

## Тени

Тени заметно улучшают объём и читаемость сцены, но увеличивают нагрузку на видеокарту.

### Enable shadows

Глобально включает тени в сцене.

Важно понимать разницу:

- этот параметр включает поддержку теней на уровне всей панели;
- отдельно у источников света и у объектов тоже должны быть включены параметры, связанные с тенями.

Если глобальные тени выключены, локальные настройки теней у источников света и объектов не дадут эффекта.

### Shadow type

Алгоритм рендеринга теней.

#### basic

Самый простой и быстрый вариант. Подходит, когда важна производительность.

#### percentage

Более сглаженные края теней. Хороший базовый вариант для большинства сцен.

#### soft

Более мягкие тени с приятным визуальным результатом.

#### variance

Самые мягкие и тяжёлые по ресурсам тени. Подходят для сцен, где качество важнее производительности.

## Фон панели

### Change background color

Позволяет использовать однотонный фон панели.

Этот параметр доступен, когда отключён фоновый рисунок.

### Background color

Цвет фона панели.

### Use background image

Вместо однотонного фона использует изображение.

Этот параметр доступен, когда отключён однотонный фон.

### Image path

Путь к изображению, которое будет использоваться как фон.

Практический совет:

- используйте изображения умеренного размера;
- если фон отвлекает от сцены, лучше сделать его более нейтральным;
- фон с большим количеством мелких деталей может ухудшать читаемость текста и элементов.

## Качество изображения и производительность

### Antialiasing (MSAA)

Сглаживание краёв объектов.

Что даёт:

- делает границы объектов визуально чище;
- особенно заметно на диагональных линиях и тонких деталях.

Что важно:

- улучшает качество картинки;
- может заметно снизить FPS на слабых устройствах;
- особенно влияет на производительность на экранах с высокой плотностью пикселей.

### GPU power preference

Предпочтительный режим работы видеокарты.

Обычно доступны варианты:

- **Default** — стандартное поведение браузера и системы;
- **High performance** — приоритет производительности;
- **Low power** — приоритет экономии энергии.

На ноутбуках с двумя видеокартами режим высокой производительности может дать заметный прирост FPS.

## Dynamic DPR

DPR — это внутреннее разрешение рендеринга относительно устройства.

### Dynamic DPR

При движении камеры качество временно снижается, а после остановки снова повышается. Это помогает сделать навигацию по сцене более плавной.

### DPR while moving

Коэффициент качества во время движения камеры.

Меньшее значение:

- повышает производительность;
- делает картинку менее чёткой во время движения.

### DPR at rest

Коэффициент качества, когда камера не двигается.

Большее значение:

- улучшает чёткость;
- повышает нагрузку на GPU.

## Adaptive DPR (auto)

Автоматически снижает качество, если падает FPS, и повышает его обратно, когда производительность восстанавливается.

Это хороший вариант для сцен с непредсказуемой нагрузкой.

### Adaptive DPR minimum factor

Минимальный предел, до которого система может понизить качество.

Чем ниже значение, тем сильнее панель готова жертвовать детализацией ради плавности.

## Add debug

Показывает служебную информацию поверх сцены. Полезно для настройки производительности.

Обычно отображаются:

- **fps** — сколько кадров в секунду рисуется;
- **ms** — сколько времени занимает один кадр;
- **memory** — примерный объём используемой памяти;
- **calls** — количество draw calls;
- **triangles** — количество треугольников в сцене;
- **geometries** — число геометрий;

Чем меньше draw calls, лишних треугольников и тяжёлых моделей, тем легче сцена для браузера.



## Add control elements

Добавляет интерактивные элементы управления трансформацией объектов прямо в сцене.

Это нужно, если вы хотите двигать, поворачивать и масштабировать объекты не только через поля редактора, но и мышью.

### Size of control elements

Размер интерактивных манипуляторов.

Больший размер удобнее в больших сценах или на экранах высокого разрешения.



## Практические рекомендации

### Если у вас тормозит сцена

Попробуйте по порядку:

1. отключить тени;
2. отключить сглаживание;
3. уменьшить DPR;
4. отключить debug и лишние служебные элементы;
5. упростить модели.

### Если нужна красивая картинка

Включайте улучшения постепенно:

1. сначала подберите свет;
2. потом включите тени;
3. затем поднимите DPR;
4. в последнюю очередь добавляйте тяжёлые фоновые изображения и сложные модели.
