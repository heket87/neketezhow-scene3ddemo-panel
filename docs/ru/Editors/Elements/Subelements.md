# Subelements

Subelements — это отдельные части загруженной модели. Они появляются после включения **Decompose object**.

Если модель состоит из нескольких мешей, вы сможете управлять каждой частью отдельно.

## Когда это нужно

- покрасить разные части модели по-разному;
- скрывать и показывать отдельные детали;
- делать кликабельной только часть объекта;
- добавлять подписи только к конкретным деталям;
- менять прозрачность по частям.

## Что такое часть модели

Это отдельный фрагмент геометрии, найденный внутри файла модели. Обычно он получает своё имя автоматически.

### Subelement name

Имя части модели.

Обычно оно приходит из исходной структуры модели и служит для ориентации в списке.

## Общие параметры части модели

### Exclude from hover

Исключает часть из реакции на наведение.

Это полезно, если деталь не должна перехватывать hover-события.

### Render on both sides

Делает материал части двусторонним.

### Cast shadows / Receive shadows

Настройка теней отдельно для этой части.

### Visibility

Позволяет управлять видимостью детали по данным или вручную.

Принцип такой же, как у обычного элемента.

### Make element clickable

Позволяет открыть ссылку по клику на эту часть модели.

### Add text window

Привязывает всплывающее текстовое окно именно к этой детали.

### Element transparency

Прозрачность отдельной части.

## Цвет части модели

### Change element color

Включает управление цветом этой детали.

### Use query for element color

Связывает цвет с данными.

### Use gradient for element color

Включает окраску по общему градиенту панели.

### Element color

Обычный статический цвет детали.

## Анимации цвета у части модели

### Add animation to element

Позволяет добавить цветовую анимацию именно для этой части.

Это удобно, если в модели есть отдельные индикаторы, лампы, кнопки или участки, которые должны менять цвет независимо от остального объекта.

## Подсветка при наведении

Если у исходной модели включена подсветка частей при наведении, она помогает быстрее понять, какая именно деталь сейчас выбрана.

## Практический совет

Разбирайте модель на части только тогда, когда это действительно нужно. Для простой статической модели без особой логики это может быть лишним усложнением. Но для интерактивных объектов такой режим очень полезен.
